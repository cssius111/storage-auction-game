# 持久化

## P0 原则

优先使用简单的本地保存格式或轻量数据库，不提前引入分布式系统。保存结构必须版本化、可校验、可迁移。

## 保存内容

- 游戏层级与当前状态
- 内容、schema 和规则版本
- 根种子与随机算法标识
- 已生成的仓库真相
- 每席位观察、信念和估值快照
- 拍卖领域事件
- 账本与记分板
- AI 调用元数据，但不保存敏感正文

## 不保存

- DeepSeek API 密钥
- 原始完整模型提示词
- 模型思维链
- 浏览器会话或认证令牌

## 保存时机

- Episode 建立后
- 仓库生成后
- 观察或估值锁定后
- 每次有效拍卖状态转换后
- 成交、揭晓、鉴定和结算后

## 一致性

- 成交事件、所有权转移和账本扣款必须原子写入。
- 保存失败不得让内存状态继续假装成功。
- 加载时先验证 schema 和版本，再构造领域对象。
- 未知字段默认拒绝，迁移器显式处理历史版本。

## 回放

回放从保存的固定真相与领域事件重建状态，不重新调用随机源、真实时钟或 AI 服务。
