# 执行计划

## 当前目标

建立一个可执行、可审查的阶段一文档基线。在文档与关键决策冻结前，不实现完整游戏。

## Task 0 退出条件

- 产品范围明确列出包含项与排除项。
- 领域层、信息层、状态机和金额语义没有相互冲突。
- DeepSeek 请求、响应、超时、重试、熔断和回退策略已写清。
- ADR 覆盖已确认的关键选择。
- `.env.example` 不包含真实密钥，`.gitignore` 会排除本地敏感文件。
- 文档链接与术语通过检查。

## P0 质量门槛

- 普通开发检查至少运行 200 个种子。
- P0 评审前至少运行 2,000 个种子。
- 同一内容版本与同一种子产生相同仓库真相。
- 断网或模型故障不阻断游戏。
- 玩家客户端和任一 AI 席位都无法获得未授权真相。
- 余额恒等式、出价合法性和账本不可绕过。

## 后续最近一步

Task 0 完成后，只进入 Task 1：建立最小 Python 工程、测试配置和空的领域边界；不提前实现拍卖玩法。
