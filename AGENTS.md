# AGENTS.md

本文件约束所有在本仓库内工作的编码代理与协作者。

## 开工前

1. 阅读本文件、`PLANS.md`、相关产品文档、架构文档与 ADR。
2. 确认当前任务范围，不提前实现未来阶段。
3. 保留用户已有修改；只改与任务直接相关的文件。
4. 若需求与已接受 ADR 冲突，先提出冲突并记录新决策。

## 语言与命名

- 面向用户的说明和产品文档使用中文。
- 代码标识符、JSON 字段、数据库字段使用英文 `snake_case`。
- 品牌、平台、人物和市场主体均使用虚构名称，避免真实商标依赖。

## 核心领域约束

- 物品真相在仓库生成时固定，观察只增加信息，不改变真相。
- 随机性必须通过显式注入的 `RandomSource` 和种子使用；禁止依赖全局随机状态。
- 时间、ID、AI 服务也必须可替换，分别通过 `Clock`、`IdFactory`、`AIProvider` 注入。
- 所有金额以整数美分保存；禁止使用浮点数表示货币。
- 所有余额变化必须经过 `Ledger`，不得直接修改角色现金。
- 玩家与 AI 使用同一套合法动作命令；客户端与模型都不能绕过规则。
- 服务端不得把 `TruthState` 或其他角色私有信息发给客户端。

## AI 与安全

- 每个 AI 席位使用隔离上下文，只接收该席位私有观察、公共信息和合法动作 ID。
- 模型只选择服务端提供的动作 ID，不得自由指定出价金额。
- 所有模型返回必须做结构校验；超时、非法输出或熔断时使用规则 AI 回退。
- 不记录真实密钥、完整提示词、隐藏事实、跨席位上下文或模型思维链。
- 模型文本按不可信纯文本处理，不允许直接执行或渲染为 HTML。
- CI 和自动测试不得调用真实 DeepSeek API。

## 代码结构

- 核心规则不得依赖 FastAPI、React 或 DeepSeek SDK。
- 领域层只表达规则；应用层负责流程编排；适配层负责外部服务。
- 优先小模块和清晰边界，不创建无必要的抽象层、微服务或基础设施。
- P0 不引入 Docker 编排、Prometheus、消息队列或复杂部署系统，除非新 ADR 明确批准。

## 完成标准

提交前至少完成：

1. 运行与改动相关的单元、契约和集成测试。
2. 运行格式、静态检查、类型检查和 schema 校验。
3. 检查密钥、日志与信息泄露风险。
4. 审查最终 diff，更新 `PLANS.md` 的状态和后续任务。
5. 说明已验证内容、未验证内容与剩余风险。
